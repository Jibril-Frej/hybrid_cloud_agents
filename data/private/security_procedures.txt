ACME Corp — Security Procedures (CONFIDENTIAL — INTERNAL USE ONLY)

Access Control

All production systems require multi-factor authentication (MFA). SSH access to
production servers is restricted to the security team and must be approved by a
member of the Infrastructure Security group via the internal ticketing system.

Incident Response

Security incidents must be reported within 1 hour of detection to security@acme.internal.
The on-call security engineer can be reached at the internal pager number 7700.
All incidents are classified as P1 (critical), P2 (high), or P3 (medium) within
30 minutes of initial report.

Data Classification

RESTRICTED: Customer PII, financial records, source code.
CONFIDENTIAL: Internal reports, HR data, sales pipeline.
INTERNAL: General company information not intended for public release.
PUBLIC: Marketing materials, public documentation.

Restricted data must be encrypted at rest using AES-256 and in transit using TLS 1.3.
Any transfer of Restricted data outside company-managed infrastructure requires
written approval from the Chief Information Security Officer.

Vulnerability Disclosure

Internal vulnerability disclosures must be submitted to the security team via the
internal VPN-accessible portal at https://secops.acme.internal/disclose.
Do not discuss vulnerabilities in public channels or unencrypted communications.
